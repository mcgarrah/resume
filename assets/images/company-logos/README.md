## Usage Notes

### Preferred Formats (best to worst)
1. **SVG** — scalable, smallest file, crisp at any size
2. **PNG** — good for logos with transparency or complex gradients
3. **JPG** — avoid for logos; compression artifacts visible at small sizes

### Resume Display Size
Logos render at 48px in the resume layout. Files under 3KB are likely too
low-resolution to look good at that size. Use `preview.html` to visually verify.

### Dark Mode Considerations
- SVG logos with dark fills may need a light background container in dark mode
- The Georgia Tech logo (navy/gold) works well on both light and dark
- The NC LIVE dogwood logo (green/blue) works on both
- The ZD mark (black) needs a light container in dark mode
- NC DIT and NC DOR logos have dark text — need light container in dark mode

### File Naming Convention
- Lowercase with underscores: `company_name_logo.ext`
- Older files use hyphens — either convention is fine but prefer underscores going forward

## Lessons Learned

### Cropping SVG Logos with viewBox (The "Zoom In" Technique)

When an SVG logo has a recognizable icon/mark on one side and unreadable text on the other, you can create a cropped version by adjusting the `viewBox` to show only the icon portion. This is the same technique used for USPS eagle, MI mark, and AKC shield.

**Steps:**
1. Open the original SVG and note its `viewBox` (e.g., `0 0 298.6 37.4`)
2. Identify the icon's coordinate boundaries by examining path `d="M<x> <y>..."` values, polygon `points`, and rect `x`/`y` attributes
3. Create a new SVG with a cropped `viewBox` that frames just the icon area
4. Copy only the relevant elements (paths, polygons, rects) into the new file
5. Set `width` and `height` to `200` for consistent square display

**Example — USPS Eagle:**
```xml
<!-- Original: viewBox="1.116 1.453 179.669 27.605" (eagle + text) -->
<!-- Cropped:  viewBox="-1 -1 52 32" (eagle only, left portion) -->
<svg viewBox="-1 -1 52 32" width="200" height="200">
  <!-- Only include path data for the eagle shape -->
</svg>
```

**Example — Measurement Inc Mark:**
```xml
<!-- Original: viewBox="0 0 298.6 37.4" (M + I + caliper + "MEASUREMENT" text) -->
<!-- Cropped:  viewBox="-2 -2 98 41" (M + I + caliper symbol only) -->
<svg viewBox="-2 -2 98 41" width="200" height="200">
  <!-- polygon for M, rect for I, paths for caliper -->
</svg>
```

**Example — AKC Shield:**
```xml
<!-- Original: viewBox="0 0 427.66 654.334" width="260" height="100" (shield + text) -->
<!-- Cropped:  viewBox="-10 0 190 654.334" width="200" height="200" (shield only) -->
<!-- For AKC, the full SVG was copied and only viewBox/dimensions changed -->
<!-- because the paths use transform matrices that are hard to extract -->
```

**Finding the crop boundary:**
- Look for gaps in x-coordinates between icon and text elements
- Use `grep` or Python to extract all `M<x>` values from path `d` attributes
- For transformed SVGs: apply the transform matrix to find viewBox-space coordinates
- Add 2-5 units of padding around the icon for breathing room

**Files created with this technique:**

|------|--------|------|
| `usps_eagle_logo.svg` | `usps_logo.svg` | viewBox `-1 -1 52 32` — eagle only |
| `measurement_inc_mark.svg` | `measurement_incorporated_logo.svg` | viewBox `-2 -2 98 41` — M + I + caliper |

### AKC Shield — Render-and-Crop (When viewBox Fails)

The AKC SVG uses a complex transform matrix (`matrix(9.37,0,0,9.37,-2975,-5501)`) that places content at absolute positions regardless of viewBox. The SVG also has `style="overflow:visible"` which tells browsers to render content outside the viewBox boundaries. This combination makes the viewBox cropping technique ineffective — changing the viewBox just changes the "camera" but the content still renders at the same absolute positions.

**What was tried and failed:**
1. Adjusting `viewBox` to crop to left portion — content still rendered at same positions
2. Setting `overflow:hidden` — didn't help because the transform places content within the viewBox anyway
3. SVG-in-SVG with `<image>` and `clipPath` — browser didn't render the nested SVG reference

**What worked:** Render the full SVG to a high-resolution PNG using `cairosvg`, then crop the raster image with Pillow:

```python
import cairosvg, io
from PIL import Image

# Render SVG to PNG at high resolution
png_data = cairosvg.svg2png(url='american_kennel_club_logo.svg', output_width=780, output_height=300)
img = Image.open(io.BytesIO(png_data)).convert('RGBA')

# Find content bounds (non-transparent pixels)
# ... (scan for alpha > 10)

# Crop to left 38% of content width (the shield portion)
shield_right = content_left + int(content_width * 0.38)
shield = img.crop((content_left, content_top, shield_right, content_bottom))
shield.save('akc_shield_v2.png', 'PNG', optimize=True)
```

**Requirements:** `pip install cairosvg` (added to `.venv`)

### Clearbit Logo API — Defunct
The Clearbit logo API (`https://logo.clearbit.com/<domain>`) was a convenient
free service for pulling company logos by domain. It no longer resolves — Clearbit
was acquired by HubSpot and the public logo API has been shut down.

### Wikimedia Commons — Best Source for Historical Logos
For defunct or historical companies, Wikimedia Commons is the best source.
Use the API to get correct download URLs:
```
https://commons.wikimedia.org/w/api.php?action=query&titles=File:FILENAME&prop=imageinfo&iiprop=url&format=json
```
Direct URL guessing often fails because the hash path changes between file versions.

### NC State Agency Logos
NC state agencies use a consistent branding pattern available at `files.nc.gov`:
- `https://files.nc.gov/dit/dit-logo_0.png` (NC DIT)
- `https://files.nc.gov/dor/dor-logo.png` (NC DOR)
Other agencies likely follow the same pattern: `files.nc.gov/<agency-code>/<agency>-logo.png`

### LinkedIn Logos — Not Downloadable
LinkedIn company and university logos are proprietary assets. They cannot be
downloaded or redistributed.

### Q+E Software — Lost to History
Q+E Software (Raleigh, NC) was acquired by Intersolv in 1994 for $34M. The brand
was absorbed into Intersolv's DataDirect product line, which later became part of
Merant, then Serena, then Micro Focus. No digital logo assets survive online.
The only path to recovering the logo would be scanning physical materials
(business cards, software boxes, manuals).
